Steps

  1. If the correlation and fringe-fitting process was not performed by Dynob, do the following steps first.
  2. Check if the folder (permission at least 775, user www-data) Dynob_root_dir/tmp/sessionname exists. Create it if not.
  3. Move the correlation data (fringe files) to vc0:~/correlations/au/sessionname/sessioncode/, remove any existing alist and control files from the folder. Make sure only the 'scan folders' remain.
  4. Copy the session's VEX file to Dynob_root_dir/tmp/sessionname/sessionname.vex.
  5. Check if the sessionname.skd exists under Dynob_root_dir/scheduling/skd/sessionname.skd
  6. Check if the session index exists on the dynob > Correlation database. Create a new record if not.
  7. Then, run http://dynob.tasphys.ga/pmonitor/monitor.php?exp=sessionname. Start with this step if the fringe-fitting process was done by Dynob. This step will re-estimate the antenna SEFDs and store them onto the database
  8. Finally, run http://dynob.tasphys.ga/pmonitor/reflux.php?exp=sessionname to monitor the flux density for sources observed in the session.
