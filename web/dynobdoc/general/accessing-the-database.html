Databases

There are two databases used by Dynob: dynob and mfringe. The dynob database stores information of the 24-hour AUSTRAL sessions, whereas the mfringe database stores results of the fringe check sessions.

dynob's structure

This database contains 5 tables:
Correlation Index of sessions, contains the session names, prefixes and the corresponding 4 digit codes.
Fluxmons Re-calculated S-band soure flux densities for observed sources. The number of columns increases automatically when new sources are monitored.
Fluxmonx Re-calculated X-band soure flux densities for observed sources. The number of columns increases automatically when new sources are monitored.
RaidList The list of computers that potentially store the sessions' data.
SEFDmon Re-estimated SEFDs for the AUSTRAL telescopes. Currently supports Hb, Ke, Yg, Ww and Ht. If more stations are involved, please add the new station's Mk4 code on the global configuration file at the $stcodes array. Columns are automatically added when new stations are monitored.

mfringe's structure

This database contains only 1 table, i.e., SEFDmon, as it assumes all data are located on the main correlator machine. This table is similar to the SEFDmon table for dynob.
