Overview

There are five extra actions available after the fringe-fitting process, independent of te Dynob's performance monitoring and feedback system. This function allows the creating of level1 vgosDB semi-automatically by runnig executing five URIs separately. Oaf and ops7 are the default for preparing the analysis and creating the vgosDB. Please change the common/global.phpsettings if using other machines. IMPORTANT: this script will only work if the entire session was processed by Dynob from correlation to fringe-fitting, except for action 1.

Actions

All automation for this part is handled by the dynob.tasphys.ga/createdb/lv1db.php?exper=sessionname&do=action. Replace the sessionname with the session to process, and the action with the following. Please run the steps in correct order unless some of the steps have been done manually.
Order action description
1 checkdir This action checks and creates the required directories on the oaf machine. This step can be skipped if the directories exists.
2 cp2mac This action copies the required files for analysis to the oaf machine. This step can be skipped if all required files were previously prepared.
3 data2mac This action copies the correlated data (fringe files) from the correlator to the oaf machine. This process can take longer for larger sessions with more scans. This step can be skipped if the fringe files have been transferred previously. This action can be run before the cp2mac.
4 predb This action generates the files required to generate vgosDB. The process can take a while. This step can be skipped if the ovex files and etc had been prepared, or if creating these files manually is preferred.
5 createdb This action generates the vgosDB.
